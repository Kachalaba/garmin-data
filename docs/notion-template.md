# 🗒️ Notion template для дайджестів

Рутини з [`routines/`](./routines/) пишуть щоденні й тижневі дайджести як **дочірні сторінки** однієї parent-сторінки у Notion. Ця parent-сторінка і є "шаблоном" — її треба створити один раз перед тим, як реєструвати scheduled-tasks.

---

## 📐 Структура parent-сторінки

Мінімальний набір, який очікують рутини:

1. **Архівна таблиця** (заголовок `## 📋 Архів дайджестів`) — щоденна рутина додає в неї по рядку після кожного запуску.
2. (Опційно) **Контекст / фази** — будь-яка таблиця, де ти вручну відмічаєш періоди життя, які впливають на readiness (відрядження, хвороба, нова робота тощо). Рутини її **не оновлюють**, але Claude може читати її при формуванні дайджесту, якщо попросиш.
3. (Опційно) **Технічний стек / Як запустити** — довідкові секції для тебе самого, коли повертаєшся до проєкту через пів року.

Дочірні сторінки (повні дайджести) рутини створюють автоматично — **нічого руками заздалегідь там не треба**.

---

## 📋 Схема архівної таблиці

6 колонок, саме у такому порядку — саме так у щоденний prompt зашита інструкція "додай рядок":

| Колонка | Тип | Приклад | Звідки береться |
|---------|-----|---------|-----------------|
| Дата    | date / text | `2026-04-17` | сьогоднішня дата |
| Readiness | text | `52 HIGH` | `training_readiness_score` + `training_readiness_level` |
| HRV     | number / text | `68` або `—` | `hrv_last_night_avg` |
| rHR     | number | `48` | `resting_heart_rate` |
| Сон     | text | `5.4г` | `sleep_duration_hours` |
| Рекомендація | select / text | `ТРЕНУВАТИСЬ` / `ЛЕГКО` / `ВІДНОВЛЕННЯ` | висновок дайджесту |

> Якщо хочеш — в Notion можна зробити колонку `Рекомендація` типу **Select** із трьома опціями й кольорами (зелений/жовтий/червоний). Рутина цього не вимагає — пише просто текст, — але для очі зручно.

## 🧱 Мінімальний markdown для вставки в Notion

Створи у себе в Notion нову сторінку (наприклад `Daily Health Digest`), і встав туди цей блок — далі Notion сам розкладе його на блоки:

```markdown
# 🏥 Daily Health Digest

Parent-сторінка для щоденних та тижневих health-дайджестів від Claude-агента.
Повні дайджести — у дочірніх сторінках (автоматично створюються рутиною).

## 📋 Архів дайджестів

| Дата | Readiness | HRV | rHR | Сон | Рекомендація |
|------|-----------|-----|-----|-----|--------------|
|      |           |     |     |     |              |

## 📊 Контекст (фази)

| Період | Що відбувається | Вплив на readiness |
|--------|-----------------|--------------------|
|        |                 |                    |

## ⚙️ Технічний стек

- Garmin Venu 3 → garmy → SQLite `health.db`
- Аналітика: `analytics/run_all.py` (HRV baseline, RHR anomaly, weather)
- Claude + MCP: `garmy`, `intervals-icu`, `notion`, `scheduled-tasks`
- Grafana дашборд `http://localhost:3000` (read-only)
- Репо: https://github.com/Kachalaba/garmin-data
```

Порожній рядок у таблиці — просто щоб рутині було куди "дописувати" новий рядок. Після першого запуску він зникне під свіжим записом.

## 🔗 Де взяти URL для `{{NOTION_DIGEST_PARENT_URL}}`

1. У Notion відкрий створену сторінку.
2. `Share` → `Copy link` (або правий клік на сторінку в sidebar → Copy link).
3. URL виглядатиме як `https://notion.so/your-workspace/Daily-Health-Digest-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`.
4. Встав цей URL у плейсхолдер `{{NOTION_DIGEST_PARENT_URL}}` у `routines/morning-digest.md` та `routines/weekly-summary.md` перед реєстрацією scheduled-tasks.

## 🤖 MCP-доступ до Notion

Щоб Claude міг писати у цю сторінку, має бути підключений `notion` MCP-сервер:

1. У Notion: **Settings & Members → Connections → Develop or manage integrations**.
2. Створити internal integration (readonly ❌ — потрібен write).
3. Відкрити parent-сторінку → `⋯` → **Connections** → додати свою integration.
4. Скопіювати integration token у конфіг Claude MCP (див. [`setup.md`](./setup.md)).

Без цього кроку рутина впаде на кроці "Створити підсторінку" — Notion API відповість `unauthorized`.

## 🧪 Перевірка

Після налаштування запусти morning-digest **вручну** один раз (не чекаючи cron):

```
У Claude: "Run morning-health-digest prompt now"
```

Очікувано:
- в архівній таблиці з'явиться один новий рядок;
- під parent-сторінкою з'явиться дочірня `Digest YYYY-MM-DD` з повним текстом дайджесту.

Якщо рядок не додався, але дочірня є — найчастіше проблема в тому, що Notion API не знаходить таблицю (рутина шукає її за заголовком `## 📋 Архів дайджестів` — перевір назву).
